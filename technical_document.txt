# Documentación Técnica: Variables de Entorno

## Cambio Realizado
Se ha implementado el uso de variables de entorno para manejar credenciales sensibles (`PHONE` y `API_KEY`) en `server.js`.

## Motivo
Evitar la exposición de datos sensibles en el repositorio de código (GitHub).

## Implementación
1.  **Librería `dotenv`**: Se utiliza para cargar las variables desde un archivo `.env` en `process.env`.
2.  **Archivo `.env`**: Contiene los valores reales de `PHONE` y `API_KEY`. Este archivo está excluido del control de versiones mediante `.gitignore`.
3.  **Código**: En `server.js`, las constantes ahora se inicializan con `process.env.PHONE` y `process.env.API_KEY`.

## Cómo Debuggear
Si la aplicación falla al enviar mensajes o no conecta:
1.  Verificar que el archivo `.env` exista en la raíz del proyecto.
2.  Verificar que el archivo `.env` tenga el formato correcto:
    ```
    PHONE=tu_numero
    API_KEY=tu_api_key
    ```
3.  Asegurarse de que no haya espacios extra alrededor del signo `=`.
4.  Reiniciar el servidor después de cualquier cambio en `.env`.
