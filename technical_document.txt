# Documentación Técnica: Tienda Kosher

## 1. Visión General del Proyecto
Este proyecto es un MVP (Producto Mínimo Viable) de una plataforma de comercio electrónico diseñada específicamente para una tienda de productos Kosher. El objetivo es ofrecer una experiencia de usuario premium, confiable y tradicional, permitiendo a los clientes navegar por un catálogo, gestionar un carrito de compras y finalizar pedidos mediante notificaciones directas a WhatsApp.

## 2. Arquitectura del Sistema

### 2.1 Backend (Node.js & Express)
El servidor actúa como una API ligera y servidor de archivos estáticos.
*   **Tecnologías**: Node.js, Express.
*   **Responsabilidades**:
    *   Servir la aplicación frontend (archivos estáticos en `/public`).
    *   Proveer datos de productos (`GET /api/products`).
    *   Procesar órdenes y notificar al vendedor (`POST /api/order`).
*   **Integración Externa**: Utiliza la API de **CallMeBot** para enviar los detalles del pedido al WhatsApp del administrador de la tienda.

### 2.2 Frontend (Vanilla JavaScript)
La interfaz es una Single Page Application (SPA) construida sin frameworks pesados para maximizar el rendimiento y control.
*   **Tecnologías**: HTML5 Semántico, CSS3 Moderno, JavaScript ES6+.
*   **Gestión de Estado**: El estado del carrito se maneja localmente en el cliente y se persiste utilizando `localStorage` para mantener la sesión del usuario entre recargas.

## 4. Detalles de Implementación y Diseño

### 4.1 Identidad Visual y UI/UX
El diseño se centra en transmitir "Tradición, Confianza y Calidad Premium", alejándose de la estética genérica.
*   **Paleta de Colores**:
    *   `--primary-color` (#1a3c5e): Azul Profundo, evocando seriedad y tradición.
    *   `--accent-color` (#c5a059): Dorado, para resaltar elementos premium y llamadas a la acción.
    *   `--bg-color` (#FDFBF7): Blanco Crema, suave a la vista, evitando el blanco clínico.
*   **Tipografía**:
    *   *Títulos*: 'Playfair Display' (Serif) para elegancia y encabezados.
    *   *Cuerpo*: 'Outfit' (Sans-serif) para legibilidad moderna en textos largos.

### 4.2 Flujo de Datos del Pedido
1.  **Selección**: El usuario añade productos al carrito (array en memoria + `localStorage`).
2.  **Checkout**: Al confirmar, el frontend envía un payload JSON al backend con los items y el total.
3.  **Procesamiento**: El backend recibe la orden, formatea un mensaje legible (e.g., "Nueva Orden: 2x Jalá, 1x Vino...") y lo codifica para URL.
4.  **Notificación**: El backend invoca la API de CallMeBot.
5.  **Confirmación**: Si la API responde `200 OK`, el frontend limpia el carrito y muestra un mensaje de éxito.

### 4.3 Responsividad
El sitio adopta un enfoque *Mobile-First*, adaptándose fluidamente a diferentes dispositivos:
*   **Navegación**: Menú hamburguesa en móviles que se despliega lateralmente; barra de navegación completa en escritorio.
*   **Grillas**: Layout de productos que pasa de 1 columna en móviles a múltiples columnas en pantallas grandes usando CSS Grid (`repeat(auto-fill, ...)`).

## 5. Configuración y Despliegue

### Requisitos Previos
*   Node.js instalado.
*   Cuenta y API Key de CallMeBot (configurada en `server.js`).

### Instalación
```bash
npm install
```

### Ejecución
```bash
# Modo desarrollo (o producción simple)
npm start
```
El servidor iniciará por defecto en el puerto 3000 (o el definido en `PORT`).
